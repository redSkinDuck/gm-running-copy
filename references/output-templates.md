# Output Templates

Generate only the files that fit the user's request and the module's actual needs.

Use numbered filenames in `02-analysis/` so the GM can consume them in order:

```text
01-GM-Brief.md
02-Reading-Guide.md
03-Scene-Flow.md
04-Timeline.md
05-Situation-Board.md
06-Clue-Matrix.md
07-NPC-Cards.md
08-Location-Cards.md
09-Risk-Map.md
```

Skip files that do not fit the module, but keep the remaining filenames' numbers stable. For example, if a situation board is unnecessary, do not rename `06-Clue-Matrix.md` to `05-Clue-Matrix.md`.

## 01-GM-Brief.md

Purpose: one-page GM orientation.

Include:

- Module title
- System and expected play length
- Core premise
- Player-facing hook
- GM-only truth
- Main emotional, genre, or fantasy promise
- Main structural promise
- Critical NPCs
- Critical locations
- Critical clues
- Likely table risks
- What to read carefully in the original

## 02-Reading-Guide.md

Purpose: tell the GM how to read the original efficiently.

Include:

- Read first
- Skim
- Can ignore until needed
- Must understand before session 1
- Spoilers that matter
- Sections likely needing personalization

## 03-Scene-Flow.md

Purpose: map actual play flow, not just chapter order.

For each scene, include:

- Scene name
- Entry condition
- Player-facing situation
- GM purpose
- Key NPCs
- Key clues
- Possible exits
- Failure or stall handling
- Optional cuts

## 06-Clue-Matrix.md

Purpose: prevent investigation dead ends and clarify information flow.

Use columns:

- Clue
- Found at
- Points to
- How players get it
- If missed
- Is critical
- Reveal timing

## 07-NPC-Cards.md

Purpose: help the GM portray NPCs quickly.

For each NPC, include:

- Name
- Role
- Surface identity
- True role or secret
- Wants
- Fears
- Knows
- Hides
- Reaction to pressure
- Voice or manner
- Useful lines
- Clues they can provide

## 08-Location-Cards.md

Purpose: help the GM run locations as playable spaces.

For each location, include:

- Location name
- First impression
- What is actually here
- Interactable elements
- Clues
- Threats
- NPCs
- Exits
- If players over-investigate
- If players leave early

## 09-Risk-Map.md

Purpose: identify where the module may fail at the table.

Track risks such as:

- Clue bottleneck
- Overbinding PC identity
- Weak hook
- Overlong exposition
- Unclear transition
- Combat difficulty spike
- Unsupported emotional beat
- Ending depends on one fragile decision
- Player agency is too constrained
- Time-state complexity is hard to track

For each risk, include:

- Where it appears
- Why it matters
- Severity
- Suggested fix
- Whether to fix in the running copy

## Change-Log.md

Purpose: make edits traceable.

Include:

- Added GM notes
- Added read-aloud text
- Added clue support
- Added co-creation points
- Adjusted mechanics
- Suggested deletions
- Major rewrites
- Unresolved questions for the GM
