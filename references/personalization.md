# Personalization Guidance

Use this reference when the user wants a running copy or provides a previous prep version that reveals their style.

## General Principles

- Make the running copy the main artifact when the user edits directly inside source text.
- Keep notes candid and practical.
- Allow natural "prep note" language after structured markers.
- Preserve the original module's flavor unless the user asks for aggressive rewriting.
- Treat independent analysis files as navigational aids, not replacements for the running copy.

## User Style Heuristics

For this user's preferred style, consider:

- Loosen role or personality constraints that restrict player authorship too heavily.
- Add co-creation prompts for costumes, props, memories, relationships, symbols, songs, and performance details.
- Prefer emotional payoff and player expression over strict preservation of every module detail.
- Strengthen scenes that support the title, theme, or promised fantasy.
- Add read-aloud text for atmosphere-heavy scenes.
- Replace punitive dead ends with playable consequences when appropriate.
- Make hidden information and reveal timing explicit.
- Keep practical "NC handling" notes close to the passage they affect.

## Existing Prep Comparison

When given an original and an existing prep version:

1. Compare paragraph structure.
2. Identify inserted notes and rewritten sections.
3. Classify changes by function: loosen, clarify, intensify, cut, personalize, mechanize, redirect, or add table support.
4. Continue in the same style unless the user asks for a cleaner or more formal format.

Common patterns to preserve:

- Weakening overly strict PC personality bindings.
- Adding "NC handling" notes close to relevant passages.
- Adding co-creation prompts for player-facing identity details.
- Replacing rigid punitive scenes with relationship or conversation opportunities.
- Expanding atmosphere into read-aloud prose.
- Adding information-control reminders.
- Deleting or weakening elements that do not support the desired climax.
- Strengthening scenes that express the module's emotional promise.

## Running Copy Editing Rules

1. Do not modify the original file.
2. Preserve original text unless there is a clear reason to rewrite.
3. Put GM additions near the point of use.
4. Mark additions visibly with inline labels.
5. Keep the user's natural prep voice where useful.
6. Prefer actionable notes over literary analysis.
7. Add summaries only when they help table operation.
8. Do not flatten the module's original tone.
9. Do not remove authorial content silently; mark cuts as suggestions.
10. Make hidden information and reveal timing explicit.
